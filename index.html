How to find this cookie

Chrome, Edge, Brave, Opera, Vivaldi, Arc — all use the same DevTools layout.

  1. Open soundcloud.com in that browser and confirm you're signed in.
  2. Press F12 (or right-click → Inspect).
  3. Open the Application tab. If you don't see it, click » at the end of the tab strip.
  4. In the left sidebar: Storage → Cookies → https://soundcloud.com.
  5. Find the row whose Name is oauth_token. Double-click its Value cell, Ctrl+C to copy.
  6. Paste it above and hit Save token.
  1. Open soundcloud.com in Firefox and confirm you're signed in.
  2. Press F12 (or right-click → Inspect).
  3. Click the Storage tab. (May be hidden under » on narrow windows.)
  4. Expand Cookies → https://soundcloud.com.
  5. Click the row named oauth_token. The full value appears in the panel on the right — right-click → Copy.
  6. Paste it above and hit Save token.
  1. First time only: Safari → Settings → Advanced → tick Show features for web developers.
  2. Open soundcloud.com and confirm you're signed in.
  3. Press ⌥+⌘+I (or Develop → Show Web Inspector).
  4. Click the Storage tab → Cookies → soundcloud.com.
  5. Click the row named oauth_token, copy the Value.
  6. Paste it above and hit Save token.

The token usually stays valid for weeks or months — repeat this only if SC starts rejecting it.

Why credentials, not a cookie?

Spotify's audio is DRM-encrypted, so unlike SoundCloud there's no cookie that unlocks a downloadable file. We use the Spotify API purely to read the track or playlist's artist/title, then fetch the actual audio from YouTube (same fallback used for DRM-locked SoundCloud tracks).

  1. Go to developer.spotify.com/dashboard and log in.
  2. Click Create app. Give it any name and description; set the redirect URI to anything (e.g. http://localhost) — it's unused here.
  3. Open the app's Settings and copy the Client ID and Client secret into the fields above.

These read public catalogue metadata only — no access to your account or private playlists.

How to export this
  1. Install a cookies.txt exporter on the browser you're logged into YouTube with. Recommended: Get cookies.txt LOCALLY (Chromium) or cookies.txt (Firefox).
  2. Open youtube.com and confirm you're signed in.
  3. Click the extension icon → export for current site (or Copy the Netscape-format output).
  4. Paste the file contents above and hit Save cookies.

Without this, the YouTube fallback works for many tracks but hits the "not a bot" wall for the rest.
